# M8 最终验收清单

日期：2026-05-03

## 运行验收

1. 使用 Godot 4 打开 `D:\GameC\dark\dark-f`。
2. 运行主场景 `res://scenes/Main.tscn`。
3. 确认开局显示地图、玩家、HUD、行动按钮、日志面板和左侧科技面板。

## 地图验收

1. 地图由 TileMap/瓦片方式组织，后续可以继续在 Godot 编辑器中铺设。
2. 资源点、技能点、静默区、回声区、黑域机制均由配置或地图层驱动。
3. 移动格、高亮格、扫描范围、攻击范围与瓦片中心对齐。

## 回合与暗能验收

1. 玩家回合开始时屏幕中央显示“你的回合”。
2. 玩家行动后需要点击“结束回合”进入 NPC 回合。
3. 暗能会随移动、扫描、攻击、拾取技能、释放技能、升级科技而变化。
4. 等待可以恢复额外暗能。

## 信号系统验收

1. 移动会生成移动信号。
2. 扫描会以玩家为中心立即执行，并显示扫描范围和表现效果。
3. 攻击会生成攻击信号，并有命中/未命中反馈。
4. 静默区不留下移动轨迹。
5. 回声区会生成公开广播信号。

## 技能与科技验收

1. 移动到技能点后可以拾取并打开三选一技能。
2. 技能进入底部技能槽，最多保留 3 个。
3. 技能释放前显示范围，区域技能显示影响范围。
4. 技能释放后进入冷却，冷却中技能按钮禁用。
5. 侦察、移动、隐匿科技可以升级，并改变扫描范围、移动范围或信号强度。

## NPC AI 验收

1. NPC 默认隐藏，普通模式下只在玩家附近暴露。
2. NPC 回合逐个行动，不会一次性全部走完。
3. NPC 移动时显示移动轨迹和落点反馈。
4. NPC AI 不读取玩家真实坐标，只依据可见信号、公开信号、地图点、黑域和自身可行动格决策。
5. NPC 会优先脱离黑域，再追踪信号，再争夺地图点，最后巡逻。

## GM 验收

1. 左侧科技面板有 GM 开关。
2. 打开后显示所有存活 NPC 的位置、编号和类型。
3. GM 开关只影响可视化，不改变 AI 决策。

## 胜负验收

1. 所有 NPC 被攻击或黑域清除后，显示胜利结算。
2. 玩家 HP 被黑域腐蚀至 0 后，显示失败结算。
3. 结算后普通操作按钮禁用。

## Web 导出验收

1. 项目使用 `gl_compatibility` 渲染路径。
2. 已提供 `export_presets.cfg`。
3. Web 导出目标为 `build/web/index.html`。
4. 后续在 Godot 编辑器中安装 Web 导出模板后，可以直接选择 Web 预设导出。
