# M8 调参与最终验收开发日志

日期：2026-05-03

## 本阶段目标

M8 是第一版收束阶段，目标是让当前 M0-M7 内容进入可验收、可导出、可继续迭代的状态。本阶段重点不再扩展新系统，而是检查数值下限、Web 导出约束、最终验收路径和自动化自检。

## 已完成内容

1. Web 导出预设：
   - 新增 `export_presets.cfg`。
   - 新增 Web 平台导出项。
   - 导出路径固定为 `build/web/index.html`，方便后续快速部署浏览器版本。

2. Web 兼容配置：
   - 保持 `gl_compatibility` 渲染路径。
   - 增加窗口伸缩配置，浏览器窗口变化时画面可扩展显示。

3. 最终自检测试：
   - 新增 `tests/m8_final_test.gd`。
   - 检查主场景路径。
   - 检查 Web 渲染器约束。
   - 检查 Web 导出预设是否存在。
   - 检查基础数值下限。
   - 检查技能池、技能槽和科技配置完整性。

4. 最终验收清单：
   - 新增 `docs/M8_最终验收清单.md`。
   - 按地图、回合、信号、技能、AI、GM、Web 导出、测试等模块列出验收项。

## 验证结果

1. M4 规则测试通过。
2. M5 规则测试通过。
3. M6 AI 测试通过。
4. M8 最终自检通过。
5. 主场景可在 Godot Headless 下启动，无脚本解析错误或节点路径错误。

## 当前结论

第一版 M0-M8 已形成闭环：玩家可以移动、扫描、攻击、采集、拾取技能、升级科技；NPC 可以基于信号和地图机制逐个行动；黑域会随回合收缩并造成阶段性伤害；GM 开关可用于验收；项目具备 Web 导出预设。

1. 使用 Godot 编辑器继续铺设真实瓦片地图。
