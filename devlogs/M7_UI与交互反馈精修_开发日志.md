# M7 UI 与交互反馈精修开发日志

日期：2026-05-02

## 本阶段目标

严格按照阶段顺序进入 M7。本阶段不改变 M6 的 NPC AI 决策规则，重点补齐玩家交互反馈，让每个主要操作都能在界面或地图上得到明确回应。

## 已完成内容

1. 动态按钮状态：
   - 玩家已行动、NPC 回合、游戏结算时，普通行动按钮会自动禁用。
   - “采集”只有站在资源点时可用。
   - “拾取”只有站在技能点且暗能足够时可用。
   - 科技按钮会根据暗能和等级上限自动启用或禁用。
   - GM 开关始终可用，方便验收。

2. 攻击范围示意：
   - 点击“攻击”后，地图会显示当前攻击范围。
   - 鼠标悬停目标格时，会额外高亮当前攻击目标格。
   - 攻击执行后清理范围示意，避免残留误导。

3. NPC 移动反馈：
   - NPC 回合仍然保持逐个行动。
   - NPC 移动时会显示短暂移动轨迹和落点反馈。
   - GM 显示开启时，可以更清楚地观察 NPC 逐个移动后的新位置。

4. 交互状态清理：
   - 移动、扫描、攻击、采集、拾取、等待等行动完成后，会清理不再适用的范围/预览层。
   - 技能冷却中的技能槽会自动禁用，减少无效点击。

## 验收方式

1. 开始玩家回合后，观察可移动格、按钮状态是否符合当前条件。
2. 不在资源点时，“采集”应不可用；移动到资源点后，“采集”变为可用。
3. 不在技能点时，“拾取”应不可用；移动到技能点且暗能足够时，“拾取”变为可用。
4. 点击“攻击”后，地图上应出现攻击范围；移动鼠标时目标格会高亮。
5. 结束回合进入 NPC 回合后，NPC 应逐个行动，并显示移动轨迹和落点反馈。
6. 打开 GM 显示后，可以看到所有存活 NPC 的当前位置、编号和类型。
